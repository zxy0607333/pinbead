## 图纸生产原则

优先做原创、几何化、低版权风险的图纸。

不要做：

- 明显受版权保护的动画角色、游戏角色、电影角色。
- 品牌 logo、商标、队徽。
- 明星、真人肖像。
- 从 Pinterest、Reddit、小红书、Etsy、Instagram 等平台直接搬运的图纸。
- 只有图片没有英文说明的薄页面。

每个图纸应该满足：

- 尺寸优先 16x16、24x24、32x32。
- 颜色数量优先 4-12 色。
- 用豆数量适合新手完成。
- 有清晰轮廓，避免大量单格噪点。
- 可打印，能实际照着拼。
- 详情页有独立英文说明，不复制模板。

## 图纸详情页内容模板

每个图纸发布时，后台字段建议按这个结构填写。

### 标题

格式：

```text
Cute Cat Bead Pattern
Tiny Strawberry Bead Pattern
Beginner Heart Keychain Bead Pattern
```

### 摘要

80-150 个英文字符，说明适合谁、尺寸、用途。

示例：

```text
A small beginner-friendly cat bead pattern with a simple outline, soft colors, and an easy printable grid.
```

### 说明正文

建议 200-400 英文词，分为 3-5 段：

1. 图纸用途：适合新手、钥匙扣、冰箱贴、礼物、节日装饰等。
2. 制作特点：尺寸、颜色数、轮廓、是否适合小板。
3. 制作提示：先拼轮廓还是先拼色块、如何检查对齐、哪些颜色可替换。
4. 打印提示：打印图纸、按行检查、准备颜色。
5. 编辑提示：可以打开到 `/editor` 修改颜色或细节。

### SEO title

格式：

```text
Cute Cat Bead Pattern - Free Printable Grid
```

### SEO description

140-160 个英文字符，包含 printable、bead pattern、尺寸或用途。

### 制作提示

每页至少包含：

- 先准备哪些颜色。
- 从哪里开始拼更稳。
- 哪些细节可以替换。
- 如果打印，如何避免看错行列。

## 发布流程

1. 在 `/admin/patterns/new` 创建草稿。
2. 用 `/editor` 画图纸，导出 PNG 预览。
3. 将 Pattern JSON、预览图、下载图上传到后台草稿。
4. 填写标题、slug、摘要、说明、分类、难度、SEO title、SEO description。
5. 检查颜色数、总用豆数和图片是否一致。
6. 确认图纸来源安全。
7. 发布为 `published`。
8. 检查 `/pattern/[slug]`、分类页、首页和 sitemap 是否正常出现。

如果手动从编辑器搬运到后台效率太低，再优先实现 `5.9 支持从编辑器保存到后台草稿`。

## 内容 QA 清单

发布前逐页检查：

- [ ] 页面不是空白页。
- [ ] H1 和 SEO title 不重复堆砌关键词。
- [ ] 英文说明是独立内容，不是复制模板。
- [ ] 图纸能看出具体形状。
- [ ] 图纸适合实际拼豆，不是图片转换噪点。
- [ ] 预览图和下载图可打开。
- [ ] 图纸来源安全。
- [ ] 内链指向真实已发布页面。
- [ ] 不在下载按钮、编辑按钮、上传按钮附近放广告。
